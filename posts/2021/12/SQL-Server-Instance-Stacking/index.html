SQL Server Instance Stacking (Multiple Instances)

  1. 說明
    1. 使用不同 Port
    2. 使用不同 IP
  2. 參考資料
  3. 相關連結

筆記 SQL Server Instance Stacking 如設定不同 Port、不同 IP 作為連線區別。

說明

使用不同 Port

需要注意防火牆有開放 1433 以及其他的 Port 以提供連線。

不同的 Port 使用，必須要在 IPALL 的 Port 上設定才能夠發揮作用。

使用不同 IP

如果是使用不同 IP 的方式，則必須要在伺服器的網路介面卡上綁上第二個 IP。

並且在組態設定上將所有的 Instance TCP/IP Listen ALL 關閉

此時不同 Instance的不同 IP 可以使用相同的 1433 Port，此時 IPALL 的 Port 設定已無意義。
