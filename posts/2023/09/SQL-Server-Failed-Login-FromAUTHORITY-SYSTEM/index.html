SQL Server | 是誰在敲打我窗 來自 NT AUTHORITY\SYSTEM 失敗登入

  1. 說明
  2. 相關連結

說明當發生 Login failed for user ‘NT AUTHORITY\SYSTEM’. 原因: 無法開啟明確指定的資料庫 ‘AdeventureWorks’。 [CLIENT: <local machine>] 的可能原因。

說明

資料庫管理人員特別重視每一個失敗的登入紀錄，而今天碰到一個特別的失敗登入。來源是資料庫伺服器本身，使用的身分是 NT AUTHORITY\SYSTEM 🤔

花了許多時間查詢，終於找到答案是來自於安裝的 agent 服務 (AzureConnectedMachineAgent)，目的是為了雲地整合進行伺服器效能監控 😮

當資料庫已經離線，但 agent 仍會嘗試登入確認健康情形，就會發生登入失敗 😥 解決的根本之道是直接刪除離線的資料庫 😎
