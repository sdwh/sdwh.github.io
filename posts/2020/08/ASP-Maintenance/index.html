ASP 網頁服務的維護指南

  1. 說明
    1. 錯誤詳細檢視
    2. 啟用上層目錄

如何面對將近 20 年歷史的 Legacy ASP Code？如果用最小的力氣可以改寫就改寫；如果用更小的力氣可以維護，那就維護吧 🤣

說明

錯誤詳細檢視

詳細的錯誤訊息能夠幫助你解決很多問題，預設情況下只有 http status code 幫助有限，啟用詳細錯誤訊息可以更快地進行除錯。

  1. 錯誤網頁 → 編輯功能設定 → 錯誤回應「詳細錯誤」
  2. ASP → 偵錯內容 → 啟用「將錯誤傳送到瀏覽器」
  3. Windows RUN → inetcpl.cpl → 進階 → 取消「顯示易懂的HTTP錯誤訊息」



啟用上層目錄

Legacy Code 常會有呼叫上層目錄檔案的寫法，因此如果移轉網頁伺服器沒有開啟這個功能的話就會發生問題。
